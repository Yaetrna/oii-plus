# oii+ v6.0.0 — Overhaul Plan

> **For Hermes:** Use `delegate_task` per phase/feature to implement this plan task-by-task.

**Goal:** Ship v6.0.0 with proper CSS, unit tests, shared helpers, and reduced runtime complexity.

**Architecture:** Keep Manifest V3, no build step, no TypeScript — stay lean. Focus on: (1) moving CSS out of JS-injected `<style>` tags into a proper loaded stylesheet, (2) adding Vitest tests for the pure math functions, (3) eliminating duplicate code between popup and content script, (4) reducing wasteful message roundtrips and MutationObservers.

**Tech Stack:** Vanilla JS (ES modules), Vitest (testing), shell script (packaging).

---

## Audit Summary (from code review)

| Area | Current State | Problem |
|------|--------------|---------|
| CSS | Injected via `ui.js:addStyles()` as JS template literal | No syntax highlighting, can't be cached by browser, breaks CSP in strict contexts |
| Tests | None | Core math (II/SI formulas, color interpolation, formatPlaytime) is untested |
| Duplicated code | `formatPlaytime` in ui.js + popup.js; `browserAPI` detection in config.js + popup.js; coefficient maps in config.js + popup.js | Inevitable across contexts, but should be minimized |
| Message passing | Popup sends **2 sequential messages** per adjustment (UPDATE_PLAYTIME → GET_CURRENT_DATA) | Extra roundtrip on every keystroke |
| Observers | `waitForInjectionPoint`, `waitForProfileData`, `setupDomObservers`, URL-change observer — 4 separate observers watching `document.body` | Redundant: `waitForProfileData` and the URL-change observer both watch `document.body` with subtree |
| Popup polling | `setInterval(checkProfile, 500)` | Wastes energy; could use storage events instead |
| Packaging | Manual `zip` command in README | No reproducible release script |
| Change tracking | Nothing | No CHANGELOG |

### Phase 3: Observer Consolidation — v6.0.0-rc

#### Task 5: Merge redundant MutationObservers

**Objective:** `waitForProfileData` (line 180) and the URL-change fallback (line 342) both observe `document.body` with `{ childList: true, subtree: true }`. Remove the URL-change fallback — Turbo events + `popstate` already cover navigation.

**Files:**
- Modify: `scripts/content.js` — remove lines 340-352 (URL-change MutationObserver)

Also refactor `waitForProfileData` to return a promise that only uses the observer + timeout (remove the polling interval — the attributeFilter on `data-initial-data` is reliable):

```javascript
function waitForProfileData(maxWaitTime = 3000) {
  return new Promise((resolve) => {
    let userData = oiiDataExtractor.getData();
    if (userData?.pp && userData?.playTimeSeconds) {
      resolve(userData);
      return;
    }

    const startTime = Date.now();
    let resolved = false;
    let pollTimer;

    const observer = new MutationObserver(() => {
      if (resolved) return;
      userData = oiiDataExtractor.getData();
      if (userData?.pp && userData?.playTimeSeconds) {
        resolved = true;
        observer.disconnect();
        clearTimeout(pollTimer);
        resolve(userData);
      }
    });

    observer.observe(document.body, {
      childList: true,
      subtree: true,
      attributes: true,
      attributeFilter: ['data-initial-data']
    });

    pollTimer = setTimeout(() => {
      if (!resolved) {
        resolved = true;
        observer.disconnect();
        resolve(oiiDataExtractor.getData());
      }
    }, maxWaitTime);
  });
}
```

**Verification:** Navigate between osu! profiles → II/SI still inject correctly. Test with browser back/forward, in-page mode switch.

## Risks & Tradeoffs

| Risk | Mitigation |
|------|-----------|
| Moving CSS to external file may cause flash of unstyled content (FOUC) | Manifest's `css:` loads before JS runs, so styles are applied instantly |
| Removing URL-change MutationObserver misses a navigation edge case | Keep it but reduce its scope — only observe `document.head` for `<title>` changes instead of full body subtree |
| Removing polling interval from `waitForProfileData` breaks on some osu! pages | `attributeFilter: ['data-initial-data']` is the exact trigger; polling was belt-and-suspenders. If issues arise, add a single `setTimeout` fallback at maxWaitTime |
| Vitest requires Node.js | Add to project docs. Windows + git-bash already installed |
| `UPDATE_PLAYTIME_AND_GET_ALL` could return stale data | injectIndices is async but the sendResponse fires synchronously. However, the update path (line 19-31) is synchronous, so by the time `injectIndices` returns, `currentUserData` is updated. The response reads the updated data. |
